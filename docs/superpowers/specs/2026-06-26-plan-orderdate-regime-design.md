# 계획 탭 — 주문날짜/경과일 표시 + 장세 태깅 — Design

## Problem

대기 목록(계획 탭)에 주문 날짜가 저장되긴 하지만, 종목코드가 있으면 카드에서 날짜가 안 보인다(`p.code || p.date` 둘 중 하나만 표시). 사용자의 매매 기준상 주문을 넣어두고 3거래일이 넘어가면 신빙성이 떨어지는데, 지금은 그걸 한눈에 확인할 방법이 없다.

또한 매매 기준의 최상위 판단(코스피가 상승장/박스장/하락장인지)을 종목 추가 시점에 같이 기록해두지 않으면, 나중에 "어떤 장세에서 한 매매가 잘 됐는지" 같은 분석을 할 수 없다.

## Goals

- 대기목록 카드에 주문날짜 + 경과 거래일(D+n)을 항상 표시하고, 3거래일 초과 시 시각적으로 강조한다.
- 종목 추가 시(계획 탭 "+ 추가") 장세(상승장/박스장/하락장) 1개를 선택하게 하고, 카드에도 배지로 표시한다.
- 장세 태그는 매매가 완료돼서 매매내역/구글시트로 넘어갈 때도 함께 보존되어, 나중에 "장세별 승률" 같은 분석이 가능하다.
- "입력" 탭(이미 매도까지 끝난 거래를 나중에 수기로 백필하는 폼)에도 동일한 장세 선택을 추가하고, 같은 시트 컬럼 위치에 값이 들어가게 한다 — 어느 경로로 기록했든 분석 시 일관되게 모인다.

## Non-Goals

- 경과일 계산에 한국 공휴일은 반영하지 않는다(주말만 제외, 명절 등은 거래일로 셈). 정교한 휴장일 캘린더 유지보수 비용이 이득보다 큼.
- "입력" 탭에는 주문날짜/D+n 개념을 추가하지 않는다 — 이미 완료된 거래를 기록하는 용도라 대기 상태가 없음.
- 장세 값은 한 번 정하면 그 plan/거래에 고정. 추가/수정 후 되돌리는 UI는 만들지 않는다.

## Data Model

`plan` 객체에 필드 추가:

- `regime` (`'up' | 'box' | 'down'`, 필수) — 화면 표시는 상승장/박스장/하락장. 추가 시점에 버튼 3개 중 하나를 선택해야 "+ 목록에 추가"가 통과된다(현재 `이름/1차매수가/수량` 필수 검증과 동일한 패턴으로 toast 경고).

`plan.date` 저장 형식 변경:

- 지금은 `new Date(dateRaw).toLocaleDateString('ko-KR')`로 `"2026. 6. 24."` 같은 한글 로케일 문자열을 저장한다. 새 plan부터는 ISO `"YYYY-MM-DD"`로 저장한다.
- 기존에 이 형식으로 시트에 이미 저장된 대기 종목 7건(테스, 피에스케이, 원익IPS, 브이엠, 한울반도체, JW신약, SK)이 있으므로, 날짜 파서(`parsePlanDate(str)`)는 ISO와 기존 `"YYYY. M. D."` 패턴(정규식 `/^(\d{4})\.\s*(\d{1,2})\.\s*(\d{1,2})\.?$/`)을 모두 인식해야 한다. 매칭 실패 시 `null` 반환하고 카드에는 D+n 없이 날짜 원문만 표시(깨지지 않게).

## 경과 거래일 계산

```js
function tradingDaysSince(dateStr) {
  const start = parsePlanDate(dateStr);
  if (!start) return null;
  let d = new Date(start), count = 0;
  const today = new Date(); today.setHours(0,0,0,0); d.setHours(0,0,0,0);
  while (d < today) {
    d.setDate(d.getDate() + 1);
    const dow = d.getDay();
    if (dow !== 0 && dow !== 6) count++;
  }
  return count;
}
```

주문일 당일은 D+0. 토/일은 카운트에서 제외. 공휴일은 셈에 포함(Non-Goals).

## UI

**대기목록 카드** — 종목코드 줄 아래에 날짜 줄을 항상 추가:

```
삼성전자
005930
주문 2026-06-24 · D+3 ⚠️
```

- `D+n`이 3 이상이면 빨간색(`--red`)으로 강조 + ⚠️ 아이콘. 2 이하는 평소 톤(`--muted`). ("3일 넘으면 신빙성 하락"을 보수적으로 해석 — D+3 시점부터 경고, D+4를 기다리지 않음)
- 장세는 카드 상단(종목명 옆 또는 바로 아래)에 작은 배지로: 상승장(초록 계열), 박스장(회색/노랑 계열), 하락장(빨강 계열).

**추가 폼("+ 추가")** — 종목명 입력 위에 케이스 버튼과 같은 스타일의 3버튼 그룹 추가:

```
[ 상승장 ]  [ 박스장 ]  [ 하락장 ]
```

아무것도 안 고르고 추가 시도 시 "장세를 선택하세요" toast, 추가 진행 안 됨.

**"입력" 탭 폼** — "케이스" 버튼 그룹 아래(또는 위)에 동일한 3버튼 그룹 추가. `saveTrade()`가 만드는 row에도 같은 위치(아래 Sheets Export 참고)에 값이 들어간다.

## Sheets Export / 연동

**`Plans` 시트 (Apps Script)**: 현재 11컬럼(id, date, name, code, b1, q1, b2, q2, custSL, sells_json, use2)에 `regime`을 12번째 컬럼으로 추가. `plan-save`/`plan-update` 둘 다 같은 위치에 써야 한다.

**메인 내역 시트 (31컬럼)**: 현재 25번째 칸이 빈 문자열로 남아있음(`finalizePlan()`과 `saveTrade()` 둘 다 동일). 이 칸에 장세 값(`상승장`/`박스장`/`하락장`)을 채운다. 컬럼 수는 그대로 31개 유지, 위치만 채워짐 — 두 저장 경로(계획→완료, 입력 탭 직접입력) 모두 같은 칸에 쓰므로 시트에서 일관되게 모인다.

`finalizePlan()`이 만드는 `hist` 객체에도 `regime` 필드를 추가(향후 매매내역 탭에서 장세별 통계를 보여줄 수 있게 — 이번 작업 범위는 아니지만 데이터는 미리 남겨둠).

## Backward Compatibility

- 기존 7건의 대기 plan: `regime` 필드가 없음(`undefined`) → 카드에 장세 배지를 표시하지 않음(빈 배지 아님, 아예 안 보임). 사용자가 직접 삭제 후 재등록하거나 그대로 둬도 에러 없음.
- 기존 plan의 `date`가 한글 로케일 문자열이어도 `parsePlanDate`가 인식해서 D+n이 정상 표시됨.
- 기존 매매내역(`hist`, localStorage)에 `regime`이 없는 항목: 화면에서 장세 정보 없이 그대로 표시(에러 없음).
