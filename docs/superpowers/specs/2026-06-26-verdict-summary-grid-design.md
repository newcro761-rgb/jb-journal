# 승/패 주관 판단 + 매매내역 요약 그리드 — Design

## Problem

지금 매매내역의 "승률"은 수익금 부호(`profit >= 0`)로만 자동 판정된다. 하지만 실제로는 수익이 났어도 전략대로 잘 실행되지 않은 거래(예: 목표가 도달 전에 불안해서 일찍 정리)와, 손실이 났어도 전략을 제대로 지킨 거래(예: 손절선까지 정확히 기다렸다가 정리)가 있다. 사용자는 이 "전략이 잘 됐는지"를 손익과 별개로 직접 판단해서 기록하고, 그 판단을 승률에 반영하고 싶다.

또한 매매내역 탭의 "최근 거래" 목록은 거래마다 종목명/상태/매도 내역/수익금/메모/수정·삭제 버튼까지 다 들어가는 세로로 긴 카드라, 9건을 한눈에 훑어보기엔 비효율적이다. 한눈에 최근 거래 9건의 종목/수익률/승패만 빠르게 훑어볼 수 있는 요약 화면이 필요하다.

## Goals

- 계획 탭 결과입력 모달에서 매도를 저장할 때 "승"/"패"를 직접 선택하게 한다(장세/케이스 선택과 동일한 필수-버튼-선택 패턴).
- 승률 통계는 이 주관적 판단이 있으면 그걸로 집계하고, 없으면(이 기능 적용 전 거래, 또는 계획 탭이 아닌 경로로 들어온 거래) 기존처럼 수익금 부호로 집계한다.
- 매매내역 탭 상단에 최근 9건을 3×3 그리드로 요약 표시한다(종목명/수익률/승패만 간단히).

## Non-Goals

- "입력" 탭(수기 백필)과 매매내역 "수정" 모달에는 승/패 선택을 추가하지 않는다 — 계획 탭 결과입력 모달에만 적용한다(사용자 확정). 이 두 경로로 기록된 거래는 승률 집계 시 수익금 부호로 폴백된다.
- 요약 그리드 셀은 탭해도 아무 동작이 없다 — 상세 확인/수정/삭제는 계속 기존 세로 리스트("최근 거래")에서 한다(사용자 확정).
- 기존 세로 리스트("최근 거래")는 손대지 않는다 — 여전히 전체 내역을 다 보여준다. 그리드는 그 위에 추가되는 별도 요약 섹션이다.
- Google Sheets 컬럼을 새로 추가하지 않는다 — 기존에 비어있는 칸 하나에 승패 라벨을 쓴다(아래 Data Model 참고).

## Data Model

### `hist.verdict` 필드

`finalizePlan()`이 만드는 `hist` 객체에 `verdict: 'win' | 'lose' | null` 추가. 계획 탭에서 저장된 거래만 값이 있고, 다른 경로(입력 탭, 수정 모달)로 만들어지거나 수정된 거래는 `null`(또는 필드 자체가 없음, 둘 다 동일하게 취급).

### 승률 계산 — 주관 판단 우선, 폴백

```js
const wins = hist.filter(h => h.verdict ? h.verdict === 'win' : h.profit > 0).length;
```

`h.verdict`가 `'win'`/`'lose'`로 설정돼 있으면 그 값을 쓰고, 없으면 기존처럼 `profit > 0`으로 판단한다.

### Sheets Export

메인 내역 시트 row(`finalizePlan()`이 만드는 31개짜리 배열)에서 진짜로 비어있는 칸은 지금 26~30번째(0-indexed)다(이전 기능에서 25번째에 장세 라벨을 채웠음). 그중 **26번째 칸**에 `'승'`/`'패'` 텍스트를 쓴다. 컬럼 개수는 그대로 유지, `Code.gs` 수정·재배포 불필요. "입력" 탭의 30개짜리 row도 같은 26번째 칸이 비어있지만, 이번 기능은 그쪽엔 손대지 않으므로 그대로 빈 채로 둔다.

## UI

### 결과입력 모달 — 승/패 버튼

"+ 매도 추가" 버튼 아래, "취소/저장하기" 버튼 위에 2버튼 그룹 추가:

```html
<div class="verdict-btns">
  <button type="button" class="verdict-btn" onclick="setVerdict('win',this)">승</button>
  <button type="button" class="verdict-btn" onclick="setVerdict('lose',this)">패</button>
</div>
```

기존 장세 버튼(`setPlanRegime`)과 동일한 필수-선택 패턴: 아무것도 안 고르고 "저장하기"를 누르면 "승/패를 선택하세요" 토스트가 뜨고 저장이 막힌다. 모달이 닫히거나(`closeModal()`) 다음 거래를 위해 다시 열릴 때(`openModal()`) 선택 상태는 초기화된다.

### 매매내역 탭 — 3×3 요약 그리드

"최근 거래" 섹션 바로 위에 새 섹션 추가. `hist` 배열은 이미 최신 거래가 맨 앞(`unshift`)이므로, 앞에서부터 최대 9개를 가져와 그리드로 그린다:

```js
const summary = hist.slice(0, 9);
```

각 셀은 종목명, 수익률(`h.rate * 100`, 기존 세로 리스트의 `hist-rate`와 같은 값), 승패 색상만 표시. 색상 판정은 승률 계산과 동일한 폴백 규칙을 그대로 쓴다:

```js
const isWin = h.verdict ? h.verdict === 'win' : h.profit >= 0;
```

`isWin`이 참이면 초록 계열, 거짓이면 빨강 계열(기존 `.green`/`.red` 클래스 재사용). 9건보다 적으면 있는 만큼만 그리드에 채워지고 빈 칸은 만들지 않는다(`grid-template-columns: repeat(3, 1fr)`이 자동으로 남는 자리를 비워둠).

## Backward Compatibility

- `verdict` 필드가 없는 기존 거래(이 기능 이전, 또는 입력 탭/수정 모달에서 온 거래)는 승률 집계와 요약 그리드 색상 모두 수익금 부호로 자연스럽게 폴백된다 — 화면이 깨지거나 빈 값이 나오지 않는다.
- 기존 결과입력 모달 사용 흐름(매도 입력 → 저장)에 승/패 선택이라는 새 필수 단계가 추가되므로, 이 기능 배포 후 첫 결과 입력부터는 승/패를 고르지 않으면 저장이 안 된다 — 의도된 동작이다.
