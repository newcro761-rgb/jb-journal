# 매도 차수별 세금 입력 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 결과 입력 모달의 매도 차수마다 세금(원) 입력을 받아서, PnL(수익금)·수익률·매매내역·Sheets row에 세금이 반영되게 한다.

**Architecture:** 기존 분할매도 기능(`docs/superpowers/plans/2026-06-18-partial-sell.md`)이 만든 `.sell-row`/`saveResult()`/`finalizePlan()`/`renderHistory()` 구조에 `tax` 필드를 추가하는 작업. 단일 파일(`index.html`) 수정, 빌드/테스트 도구 없음 — 검증은 브라우저 직접 확인(가능하면 Playwright).

**Tech Stack:** Vanilla HTML5/CSS3/JS, localStorage, Google Sheets row export(기존 30열 구조 유지).

## Global Constraints

- 변경 범위는 "계획" 탭의 결과 입력 모달 + 관련 데이터 흐름에만 한정한다. "입력" 탭의 수동 거래입력 폼은 건드리지 않는다.
- 세금은 차수(행)마다 입력, 비워두면 0. 원 금액 직접 입력만 지원(비율 자동계산 없음).
- `profit = Math.round(soldAmt - avg*totalQty - totalTax)`, `rate = profit / (avg*totalQty)` (세후 기준으로 통일).
- Sheets 30열 row의 열 개수/순서는 그대로 유지 — 21번째 칸(0-indexed, `rate` 다음에 오던 빈 칸)에 `totalTax`를 채운다.
- 레거시 매매내역 항목(`h.tax`가 `undefined`)은 화면에서 에러 없이 그대로 표시되어야 한다.
- 설계 문서: `docs/superpowers/specs/2026-06-19-sell-tax-design.md` (모든 계산식/조건은 이 문서를 따른다).

```js
  function finalizePlan(plan) {
    const use2 = plan.use2 && plan.b2 && plan.q2;
    const totalQty = use2 ? plan.q1 + plan.q2 : plan.q1;
    const avg = use2 ? (plan.b1*plan.q1 + plan.b2*plan.q2)/(plan.q1+plan.q2) : plan.b1;
    const totalAmt = plan.b1*plan.q1 + (use2 ? plan.b2*plan.q2 : 0);
    const soldAmt = plan.sells.reduce((s, x) => s + x.price*x.qty, 0);
    const profit = Math.round(soldAmt - avg*totalQty);
    const rate = parseFloat(((soldAmt/totalQty - avg) / avg).toFixed(4));
    const status = (profit >= 0 ? '수익' : '손절') + `(${plan.sells.length}차분할)`;

    const hist = loadHistory();
    hist.unshift({
      id: Date.now(), date: plan.sells[plan.sells.length - 1].date,
      name: plan.name, code: plan.code, status,
      avg: Math.round(avg), sells: plan.sells, totalQty, profit, rate
    });
    saveHistory(hist);

    const sellDateCol  = plan.sells.map(s => s.date).join(' / ');
    const sellPriceCol = plan.sells.map(s => `${s.price}(${s.qty}주)`).join(' / ');
    const b2 = use2 ? plan.b2 : 0, q2 = use2 ? plan.q2 : 0;
    const row = [
      plan.date, plan.name, plan.code,
      plan.b1, plan.q1, plan.b1*plan.q1, Math.round(plan.b1*1.03),
      b2||'', q2||'', b2&&q2 ? b2*q2 : '',
      Math.round(avg), totalAmt, Math.round(avg*0.93),
      Math.round(avg*1.03), Math.round(avg*1.05), Math.round(avg*1.07),
      Math.round(totalAmt*0.07),
      sellDateCol, sellPriceCol, profit, rate,
      '', status, '', '', '', '', '', '', '', ''
    ];
    fetch(APPS_SCRIPT_URL, { method: 'POST', mode: 'no-cors', body: JSON.stringify({ row }) });

    deletePlan(plan.id);
  }
```

- [ ] **Step 3: 브라우저에서 세금 반영 계산 확인**

1차 매수가 10000원 × 100주(2차 없음) 계획으로 결과 입력 모달 열기. 매도 행: 매도가 10500, 수량 100, 오늘 날짜, 세금 5000 입력 → 저장.

기대값 계산: `soldAmt = 10500*100 = 1,050,000`, `avg*totalQty = 10000*100 = 1,000,000`, `totalTax = 5000` → `profit = 1,050,000 - 1,000,000 - 5,000 = 45,000원`, `rate = 45000 / 1,000,000 = 0.045 (4.50%)`.

"내역" 탭에서 수익금 `+45,000원`, 수익률 `+4.50%`로 표시되는지 확인 (Task 3 완료 전에는 "세금" 줄 자체는 아직 안 보여도 정상).
